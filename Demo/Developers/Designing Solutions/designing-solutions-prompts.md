# Sample Prompts for Designing Solutions & Architecture Help

## Generate Implementation Plans
- Break down this feature into actionable steps, files to modify, and acceptance criteria.
- What are the main tasks required to implement user authentication?
- List the files and modules that need to be updated for this new API endpoint.

## Compare Approaches
- Should I use event-driven or request/response here? What are the tradeoffs?
- Which is better for this use case: microservices or a monolith?
- What are the pros and cons of using a message queue for this workflow?

## Create Scaffolds in an Existing Repo
- Add a new module/service layer following the existing codebase patterns.
- Generate boilerplate code for a new feature consistent with current architecture.
- How should I structure a new domain service in this project?

---
Use these prompts to guide solution design, architectural decisions, and consistent codebase growth.
